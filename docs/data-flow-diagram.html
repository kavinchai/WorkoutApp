1 Frontend — How Users Interact

Tech Stack

  • React 18 with Vite 5 for fast HMR and bundling
  • React Router v6 for client-side navigation between pages
  • Zustand for global state (just the JWT token & username)
  • Axios with request interceptor to auto-attach Authorization: Bearer <token>
  • Recharts for line/bar charts (weight trends, calorie tracking, strength progress)
frontend/src/main.jsx → App.jsx

Data Fetching Pattern

Every page uses a custom hook that wraps a generic useFetch(endpoint) hook. This hook calls api.get(endpoint) on mount and returns { data, loading, error, refetch }.

// hooks/useFetch.js (simplified)
const [data, setData] = useState(null);
useEffect(() => {
  api.get(endpoint)
    .then(res => setData(res.data))
    .catch(err => setError(err));
}, [endpoint]);
frontend/src/hooks/useFetch.js

API Proxy (Dev vs Prod)

Development: Vite proxies /api to http://localhost:8080 so the browser talks to the same origin.

Production: Nginx serves the static build and reverse-proxies /api requests to the Spring Boot container.

frontend/vite.config.js, frontend/Dockerfile

Zustand Auth Store

Minimal store with two fields: token and username. The login() action saves both; logout() clears them. In App.jsx, if token is null the user sees the login page; otherwise, protected routes render.

frontend/src/store/authStore.js

2 Authentication — End to End

1. User submits credentials

Login.jsx sends POST /api/auth/login (or /register) with { username, password } via Axios.

2. AuthController validates

Spring’s AuthenticationManager invokes UserDetailsServiceImpl which loads the user from PostgreSQL. Password is compared against the BCrypt hash stored in the users table.

3. JWT generated

JwtUtil.generateToken(username) creates an HMAC-SHA signed JWT with 24h expiry. The token’s subject is the username. Secret comes from JWT_SECRET env var.

4. Token stored in Zustand

Frontend receives { token, username }, calls useAuthStore.login(token, username). All subsequent Axios requests auto-attach Authorization: Bearer <token> via the request interceptor.

5. JwtFilter on every request

Before hitting any controller, JwtFilter extracts the token, validates signature + expiry, loads UserDetails, and populates SecurityContext. If invalid → 401 → Axios interceptor calls logout().

3 Backend — Layered Architecture

Controller Layer

7 REST controllers under /api/*. Each injects the authenticated user via @AuthenticationPrincipal UserDetails, ensuring users only access their own data. Request bodies are validated with @Valid.

backend/src/.../controller/

Service Layer

Business logic lives here. Examples: calculating personal records by comparing all historical sets, computing nutrition totals by summing meals, mapping entities to DTOs. Services are @Transactional so a failed multi-table write rolls back atomically.

backend/src/.../service/

Repository Layer (Spring Data JPA)

Interfaces extending JpaRepository. Spring auto-generates SQL from method names (e.g., findByUserIdOrderByLogDateAsc). Complex queries use @Query with JPQL and JOIN FETCH to avoid N+1 problems.

// WorkoutSessionRepository.java
@Query("SELECT ws FROM WorkoutSession ws " +
  "LEFT JOIN FETCH ws.exerciseSets " +
  "WHERE ws.user.id = :userId " +
  "ORDER BY ws.sessionDate ASC")
backend/src/.../repository/

DTO Pattern

Controllers never expose JPA entities directly. Instead, services map to DTOs (Data Transfer Objects) like WorkoutSessionDTO, NutritionLogDTO. This decouples the API contract from the database schema and prevents circular references during JSON serialization.

5 External Integrations

Google Calendar Sync (GitHub Actions)

Two cron-triggered GitHub Actions run daily at 5am UTC. Python scripts connect directly to PostgreSQL (bypassing the API) using psycopg2, query the day's workouts/weight, and create Google Calendar events via the Calendar API v3 with a service account.

7 Putting It All Together — Example: Logging a Workout

User clicks “Save Workout” on Today page

The WorkoutBuilderModal component collects exercises, sets, reps, and weight. It calls api.post('/workouts', payload). The Axios interceptor attaches the JWT.

JwtFilter authenticates the request

Token is extracted, validated (HMAC-SHA signature + expiry check), user is loaded into SecurityContext.

WorkoutController receives the request

@PostMapping method receives the WorkoutSessionRequest DTO. The authenticated user is injected via @AuthenticationPrincipal.

WorkoutService processes the data

Creates a WorkoutSession entity, iterates over exercises and sets, creates ExerciseSet entities. Saves via JPA cascade.

PostgreSQL persists the data

INSERT INTO workout_session + INSERT INTO exercise_set (one row per set). Both within a single transaction.

Frontend re-fetches and re-renders

After the POST succeeds, refetch() triggers GET /api/workouts. The hook updates state, React re-renders the workout list with the new entry.
